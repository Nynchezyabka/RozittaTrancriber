Превращает записи (лекции, интервью, голосовые, видео) в `.md`, которые потом
легко скармливаются в RAG (AnythingLLM, векторные
базы и т.п.).

---

## ✨ Что делает

- Берёт **все видео и аудио** из папки (mp4, mkv, avi, mov, webm, mp3, m4a, wav,
  ogg, opus, flac, aac)
- Распознаёт речь через **faster-whisper** (модели скачиваются один раз и работают локально)
- На каждый файл создаёт один `.md`:
  - первая строка `# <имя файла>` — страховка для RAG-чанков
  - таймкоды `**[MM:SS]**` (по умолчанию каждые 5 минут)
  - **опционально** разделение по ролям (диаризация) — см. ниже
- **Идемпотентность**: готовые `.md` пропускаются — можно дозапускать после сбоев
- **Crash-safe**: пишет во временный `.part`, потом атомарно переименовывает
- **Мягкая остановка** с сохранением уже сделанного
- **Прогресс скачивания модели** виден в журнале при первом запуске
- **Модели в папке проекта** (`models/`) — удалятся вместе с проектом, а не лежат
  мёртвым грузом в системном кэше
- **Звук «ква»** 🐸 при завершении батча

---

## 💡 Зачем это нужно (сценарии)

**Лекции и вебинары.** Преподаватель читает курс — у вас есть видео. Transcriber
превращает их в текст с таймкодами, а Parser собирает всё в единый корпус для
поиска: *«в какой лекции объясняли теорему Пифагора?»*

**Интервью и подкасты.** Диаризация помечает, кто говорит, — готовый расшифрованный
диалог можно загрузить в RAG и задавать вопросы по содержанию.

**Голосовые заметки и встречи.** Вместо прослушивания — сразу текст, удобно для
архивации и анализа.

## 🚀 Быстрый старт

Нужен **Python 3.10–3.12** (3.13+ не поддерживается whisperX). Скачайте или клонируйте проект, затем запустите установщик — он сам определит видеокарту:

```bash
git clone https://github.com/Nynchezyabka/RozittaTranscriber.git
cd RozittaTranscriber

install.bat            # Windows (двойной клик или из консоли)
./install.sh           # Linux / macOS
```

Установщик проверяет `nvidia-smi` и ставит нужную сборку:

| Режим | Когда | Что ставит | Итог |
|-------|-------|------------|-----|
| **GPU** | найдена NVIDIA + драйвер | torch CUDA + faster-whisper + nvidia-cublas/cudnn (~4 ГБ) | ускорение в 3–5 раз |
| **CPU** | нет NVIDIA / AMD / Intel | CPU-torch + faster-whisper, nvidia-* удаляются (~300 МБ) | работает медленнее |

Можно переопределить вручную: `install.bat --gpu` или `install.bat --cpu`
(то же для `install.sh`).

> 💡 Для CPU-сборки скрипт хитрит: сначала ставит лёгкий CPU-torch (~200 МБ),
> затем requirements, и в конце удаляет CUDA-библиотеки `nvidia-*` (~1.93 ГБ),
> которые ctranslate2 тянет как зависимость, но на CPU не нужны. Итог —
> скачано ~2 ГБ, в `.venv` осталось ~300 МБ.

После установки — запустите сервер:

```bash
python server.py       # или start.bat / ./start.sh
```

Откройте `http://localhost:8010` — интерфейс готов к работе. При первом запуске
модель `large-v3-turbo` (~1.5 ГБ) скачается в папку `models/`, в журнале будет
виден прогресс.

### Готовая сборка (для Windows)

Скачайте .exe из [Releases](https://github.com/Nynchezyabka/RozittaTranscriber/releases)
(скоро), распакуйте и запустите `transcriber.exe` — браузер откроется автоматически.
CUDA-библиотеки уже внутри, ничего ставить не нужно.

## 📖 Как пользоваться

1. Откройте веб-интерфейс (по умолчанию `http://localhost:8010`).
2. Укажите **папку с медиафайлами** (кнопка «Обзор»).
3. Выберите **папку для транскрипций** — переключатель в один из режимов:
   - **«Рядом с исходниками»** → сохраняет в `<вход>/transcripts` (по умолчанию);
   - **«Общая папка»** → сохраняет всё в одну указанную папку (удобно для RAG-корпуса).
4. Выберите модель распознавания (по умолчанию `large-v3-turbo` — хороший баланс
   качества и скорости).
5. При необходимости:
   - добавьте **подсказку** (prompt) с терминологией (имена, названия) для повышения точности;
   - включите **разделение по ролям** (диаризацию) — для интервью и диалогов.
6. Нажмите «Запустить» — в очереди отобразится прогресс, справа появится превью.
   При первом запуске модель скачается (~1.5 ГБ для `large-v3-turbo`) — в журнале
   будет виден прогресс скачивания.
7. После завершения вы можете задать имена спикерам прямо в превью — кликните
   по `SPEAKER_00` или через панель спикеров.

## 🗣️ Разделение по ролям (опционально)

Для диалогов и интервью транскрипция может быть размечена по говорящим:

```markdown
# интервью.mp3

**[00:00] SPEAKER_00:** Здравствуйте, расскажите про орхидеи.

**[00:05] SPEAKER_01:** С удовольствием. Каттлея — один из самых красивых родов...
```

**Как включить:**

- Установите галочку **«Разделять по ролям»** в интерфейсе.
- Введите ваш **HuggingFace Access Token** (бесплатно, получается на сайте).

**Подготовка (один раз):**
1. Зарегистрируйтесь на [huggingface.co](https://huggingface.co).
2. Создайте токен с правами **Read** — [настройки](https://huggingface.co/settings/tokens).
3. Примите условия использования моделей:
   - [pyannote/speaker-diarization-3.1](https://huggingface.co/pyannote/speaker-diarization-3.1)
   - [pyannote/segmentation-3.0](https://huggingface.co/pyannote/segmentation-3.0)

При первом запуске с диаризацией модель (~100 МБ) скачается в локальный кэш.
Если диаризация не нужна или токен не указан — транскрипция работает без неё.

## ❓ Частые вопросы

<details>
<summary><b>Нужен ли интернет для работы?</b></summary>
Интернет требуется только для скачивания моделей при первом запуске. После этого
всё работает полностью офлайн.
</details>

<details>
<summary><b>Куда сохраняются модели?</b></summary>
В папку <code>models/</code> рядом с проектом: Whisper (~1.5 ГБ для
<code>large-v3-turbo</code>) + pyannote (~100 МБ, если включена диаризация).
Удалятся вместе с проектом. Добавьте <code>models/</code> в <code>.gitignore</code>.
</details>

<details>
<summary><b>Какие модели используются?</b></summary>
Основная — faster-whisper (доступны модели tiny, base, small, medium, large-v2,
large-v3-turbo). Для диаризации — pyannote (скачивается отдельно).
</details>

<details>
<summary><b>Можно ли запустить без диаризации?</b></summary>
Да, просто не ставьте галочку «Разделять по ролям» — токен не потребуется.
</details>

<details>
<summary><b>Совместим ли с RozittaParser?</b></summary>
Да, готовые `.md` файлы можно скормить парсеру как обычный текст (выберите папку
с ними как источник). В семействе Rozitta это основной поток данных.
</details>

## 🐸 Семейство Rozitta

Единая экосистема инструментов для работы с текстом и знаниями — общий визуальный
стиль, взаимные ссылки, общая философия «локально и прозрачно».

| Инструмент | Статус | Назначение |
|------------|--------|------------|
| ✅ [**Rozitta Parser**](https://github.com/Nynchezyabka/RozittaParser) | готов | Парсинг Telegram-чатов, чанкинг, подготовка RAG-корпуса |
| ✅ **Rozitta Transcriber** *(этот репозиторий)* | готов | Аудио/видео → Markdown с таймкодами и диаризацией |
| 🔜 **Rozitta Библиотекарь** | в планах | Локальный поиск и вопросы по архиву |

**Поток данных:**

```
Видео/аудио ──▶ Rozitta Transcriber ──▶ .md ──▶ Rozitta Parser ──▶ RAG-корпус
                                                              │
                                                              ▼
                                                      Rozitta Библиотекарь (🔜)
```

## 👩‍💻 Для разработчиков

Ядро проекта — три файла: `server.py` (FastAPI + WebSocket), `transcribe.py`
(распознавание речи и диаризация), `static/index.html` (веб-интерфейс).
Требуется Python 3.10–3.12 (whisperX не поддерживает 3.13+).

## ⚠️ Важно

- Transcriber предназначен для **личного использования** — расшифровывайте только те записи, которые у вас есть право обрабатывать.
- Модели работают локально — ваши данные не покидают компьютер.
- Для диаризации требуется аккаунт HuggingFace и принятие условий использования.
